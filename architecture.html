System Overview

🎯 Purpose

Process insurance claims against coverage rules. Determine what's covered, calculate payable amounts, track claim lifecycle, and explain every decision.

⚡ Key Capabilities

  • Coverage rule evaluation per line item
  • Annual/lifetime limit tracking
  • Deductible application
  • Partial approvals
  • Human-readable denial explanations
  • Claim state machine with dispute flow

🏗️ Architecture

Monorepo (nx) with Express backend (InversifyJS IoC, Sequelize ORM) and Next.js frontend. Layered architecture: Controllers → Services → Repositories → Models.

State Machines

Claim Lifecycle

SUBMITTED → UNDER_REVIEW → APPROVED → PAID
↘ PARTIALLY_APPROVED → PAID
↘ DENIED
Any non-terminal → DISPUTED → UNDER_REVIEW (re-adjudicate)

Line Item States

PENDING → APPROVED (with amount + explanation)
↘ DENIED (with reason + explanation)
↘ UNDER_REVIEW (pre-auth required)

Adjudication Engine Flow

1
Policy Validation
Check is_active, verify date_of_service falls within effective_date–expiration_date. If invalid → deny entire claim with POLICY_INACTIVE.
2
Coverage Rule Lookup
For each line item, find matching CoverageRule by service_type. No match → deny with NOT_COVERED.
3
Coverage Check
If is_covered = false → deny with NOT_COVERED. If requires_pre_auth = true → flag UNDER_REVIEW.
4
Amount Calculation
approved = min(billed_amount, max_amount) × coverage_percentage
5
Limit Tracking
Sum previously approved amounts for same service_type within the limit_period. If used + approved > max_amount → approve remaining or deny with LIMIT_EXCEEDED.
6
Deductible Application
Track annual deductible consumed. Reduce approved amount by remaining deductible. Explanation shows deductible impact.
7
Explanation Generation
Every line item gets a human-readable explanation: "Consultation covered at 80% of $150.00. Approved: $120.00" or "Vision services not covered under your policy".
8
Claim Status Resolution
All approved → APPROVED. Mix → PARTIALLY_APPROVED. All denied → DENIED. Any under review → UNDER_REVIEW.
